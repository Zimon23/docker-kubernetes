 Docker Swarm은 여러 대의 Docker 호스트를 하나의 클러스터로 묶어 관리하는 컨테이너 오케스트레이션
  도구입니다. 간단히 말해, 여러 서버를 마치 하나의 서버처럼 사용하여 컨테이너를 효율적으로 관리하고
  운영하는 것을 목표로 합니다.

  주요 목적은 다음과 같습니다.

   * 클러스터링 및 확장성: 여러 Docker 서버를 하나의 가상 서버처럼 묶어 관리합니다. 이를 통해
     애플리케이션의 부하가 증가하면 간단히 서버(노드)를 추가하여 수평적으로 확장(Scale-out)할 수
     있습니다.

   * 고가용성 및 장애 복구: 클러스터 내의 특정 서버(노드)에 장애가 발생하더라도, Swarm은 다른 정상
     노드에서 해당 컨테이너를 자동으로 다시 시작하여 서비스 중단을 최소화합니다.

   * 로드 밸런싱: 여러 컨테이너에 걸쳐 들어오는 네트워크 트래픽을 자동으로 분산시켜 특정 컨테이너에
     과부하가 걸리는 것을 방지합니다.

   * 롤링 업데이트: 서비스 업데이트 시 무중단 배포를 지원합니다. 이전 버전의 컨테이너를 점진적으로
     새 버전으로 교체하여 사용자가 서비스를 중단 없이 이용할 수 있게 합니다.

   * 간편한 사용: Docker 엔진에 내장된 기능이므로 별도의 복잡한 설치 과정이 필요 없습니다. 또한,
     기존에 사용하던 Docker CLI 명령어를 그대로 활용할 수 있어 쿠버네티스(Kubernetes)에 비해 학습
     곡선이 낮고 사용이 간편합니다.

  요약하자면, Docker Swarm은 분산된 환경에서 컨테이너화된 애플리케이션을 더 쉽고 안정적으로 배포,
  확장, 관리할 수 있도록 돕는 도구입니다.
